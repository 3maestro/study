Jenkins

  1. 용어
    • Jenkins: 오픈 소스 CI/CD 도구로, 소프트웨어 개발 프로세스를 자동화하는 데 사용됩니다.
    • Pipeline: Jenkins에서 CI/CD 프로세스를 정의하고 실행하는 방법입니다. Jenkinsfile을 사용하여 파이프라인을 정의합니다.
    • Jenkinsfile: 파이프라인을 정의하는 Groovy 스크립트 파일입니다. CI/CD 파이프라인의 단계와 작업을 정의합니다.
    • Freestyle Project: Jenkins에서 구성할 수 있는 가장 일반적인 프로젝트 유형 중 하나입니다. 자유롭게 빌드 스크립트 및 환경을 구성할 수 있습니다.
    • Pipeline Project: Jenkins에서 Pipeline을 사용하여 CI/CD를 구현하는 프로젝트 유형입니다. Jenkinsfile을 사용하여 파이프라인을 정의합니다.
    • Agent: Jenkins에서 빌드나 배포 작업을 실행하는 환경을 지칭합니다. 에이전트는 마스터 노드나 다른 노드에서 실행될 수 있습니다.
    • Node: Jenkins에서 빌드나 배포 작업이 실행되는 단위입니다. 에이전트가 할당된 컴퓨터 또는 VM을 의미합니다.
    • Stage: Jenkinsfile 내에서 파이프라인의 단계를 정의하는 부분입니다. 각 스테이지는 일련의 작업을 실행할 수 있습니다.
    • Step: Jenkinsfile 내에서 각 스테이지에서 수행되는 개별 작업을 정의하는 부분입니다. 쉘 명령 실행, 빌드, 테스트 등이 포함됩니다.
    • Trigger: Jenkins에서 빌드를 시작하는 조건을 의미합니다. 예를 들어, 코드 변경이 감지되었을 때 자동으로 빌드를 시작하는 것입니다.
    • Artifact: 빌드 과정에서 생성되거나 사용되는 파일이나 데이터를 의미합니다. 보통 빌드 결과물이나 테스트 리포트와 같은 것들을 포함합니다.
    • Job: Jenkins에서 작업을 수행하는 단위입니다. 프로젝트, 파이프라인, 빌드 등을 모두 Job으로 볼 수 있습니다.
    • Plugin: Jenkins 기능을 확장하고 기능을 추가하기 위해 사용되는 소프트웨어 모듈입니다. 다양한 플러그인을 통해 Git, Slack, 이메일 통합 등을 지원합니다.
    • Slave: Jenkins 마스터 노드에 연결된 에이전트 노드를 의미합니다. 추가적인 실행 환경을 제공하여 병렬로 여러 빌드를 실행할 수 있습니다.
  2. Jenkinsfile
    • 정의
      • Jenkinsfile은 Jenkins에서 사용되는 파이프라인 스크립트 파일입니다.
      • 이 파일은 소스 코드 저장소(예: Git, SVN 등)에 함께 저장되어 CI/CD 파이프라인을 정의하고 실행하는 데 사용됩니다.
      • Jenkinsfile은 Jenkins 파이프라인의 구성을 코드로 정의하는 방법을 제공하여, 파이프라인을 버전 관리하고 재사용할 수 있도록 도와줍니다.
      • Jenkinsfile을 작성하고 사용하려면 Jenkins 서버에서 파이프라인 프로젝트를 생성하고 Jenkinsfile을 해당 프로젝트에 추가하면 됩니다. 그러면 Jenkins는 해당 Jenkinsfile을 읽고 정의된 파이프라인을 실행합니다.
    • 목적
      • 파이프라인 정의: 소프트웨어 빌드, 테스트, 배포 등 CI/CD 파이프라인의 단계를 정의합니다.
      • 버전 관리: Jenkinsfile은 코드이므로 버전 관리 시스템(Git 등)을 통해 관리될 수 있습니다.
      • 재사용성: 동일한 파이프라인을 여러 프로젝트에 재사용할 수 있습니다.
      • 자동화: Jenkinsfile에 정의된 파이프라인은 자동화되어 실행되므로, 인프라 및 배포 프로세스를 자동화할 수 있습니다.
    • 스크립트
      • Scripted Pipeline
        • Groovy 스크립트로 작성되며 유연하고 자유로운 문법을 가지고 있습니다.
      • Declarative Pipeline
        • 보다 구조화된 형태로 작성되며 가독성이 좋고 일부 기능을 단순화할 수 있습니다.
  3. Freestyle Project VS Pipeline Project
    • Freestyle Project
      • 단순한 빌드 작업이나 간단한 프로젝트에서 사용하기에 적합합니다. 초기 설정이나 프로젝트의 복잡성이 낮을 때 유용합니다.
      • 특징
        • 자유롭게 빌드 스크립트와 환경을 구성할 수 있는 가장 일반적인 프로젝트 유형입니다. 단순한 빌드 작업부터 복잡한 배포 작업까지 다양한 작업을 수행할 수 있습니다. 빌드 단계와 후속 작업을 구성하는 데 있어서 매우 유연합니다.
      • 장점
        • 사용하기 쉽고 간단하며 초기 설정이 쉽습니다. Jenkins의 다양한 플러그인과 호환되어 다양한 기능을 활용할 수 있습니다. 자유로운 설정을 통해 다양한 요구 사항에 대응할 수 있습니다.
      • 단점
        • 복잡한 파이프라인을 정의하 거나 병렬 처리를 구현하는 데에는 한계가 있습니다. Jenkinsfile에 비해 파이프라인의 코드화 및 버전 관리가 제한적입니다.
    • Pipeline Project
      • 복잡한 CI/CD 파이프라인이나 구조화된 프로젝트에서 사용하기에 적합합니다. 파이프라인을 코드로 정의하여 자동화하고 관리하고자 할 때 유용합니다.
      • 특징
        • Jenkinsfile을 사용하여 파이프라인을 코드로 정의하는 프로젝트 유형입니다. Scripted Pipeline과 Declarative Pipeline 두 가지 형식으로 정의할 수 있습니다. CI/CD 파이프라인을 코드로 관리하고 버전 관리할 수 있습니다.
      • 장점
        • 파이프라인을 코드로 정의하므로 CI/CD 프로세스를 완전히 자동화하고 버전 관리할 수 있습니다. 복잡한 파이프라인을 정의하거나 병렬 처리를 구현하는 데 유연합니다. 구조화된 문법을 통해 가독성이 뛰어나고 유지 보수가 용이합니다.
      • 단점
        • 초기 설정이 Freestyle Project에 비해 조금 복잡할 수 있습니다. Jenkinsfile을 작성하고 관리하는 데 일부 추가 작업이 필요합니다.
