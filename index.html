      • 설치시스템문의서 발송
      • JAVA, WAS, 방화벽 작업 안내
      • 패키지 설치 위치 전달 받기(필요시)
    4. 확인 전화
      • 설치시스템문의서 설명
      • 사전작업 요청
    5. 설치시스템문의서 수신
    6. 임시라이선스 발급신청
      • ERP -> 라이선스발급관리 -> 임시라이선스 발급
      • Jcaos.lic / MagicJs 라이선스 발급요청
    7. 내부 테스트
      • 설치시스템문의서 기반 구축 테스트

      • 테스트 완료된 패키지 배포 파일로 만들기
    8. 패키지 전달
      • 패키지 설치 경로 확인
      • 패키지 및 설치메뉴얼 발송
      • 전달 드린 설치메뉴얼로 설치 안내 (필요시)
    9. 설치 일정 조율
      • 원격 지원 또는 직출
      • 설치는 운용 및 개발 환경에서 작동하도록 셋팅
    10. 설치확인서 전달
      • 설치 과정 및 테스트 결과 스크린샷
      • 설치확인서 문서 작성 후 전달
      • 필요시 패키지 CD 만들어 전달
    11. 비용 정산여부 확인
      • 비용 정산에 관한 이메일 수신
    12. 정식 라이선스 요청
      • ERP -> 라이선스발급관리 -> 정식라이선스 발급
      • Jcaos.lic / MagicJs 라이선스 발급요청
      • Magic js는 도메인으로 발급 요청
      • jcaos는 ip로 발급 요청
    13. 적용 후 정식 라이선스로 테스트

					docker run \
					-i -t \
					--privileged=true \
					--name myCentOS7 \
					-v /Users/folaflor/myCentOS7home:/home \
					-p 44983:22 \
					centos:centos7
					view rawdocker_centos7.sh hosted with ❤ by GitHub
					
					옵션 설명
	
					-i -t : 상호 입출력. 터미널 사용을 위한 옵션.
	
					--privileged=true : 높은 권한으로 컨테이너 실행. ssh 외부 접속 설정을 위해서는 높은 권한 실행이 필요.
	
					--name myCentOS7 : 컨테이너 이름. 변경, 생략 가능.
	
					-v /Users/folaflor/myCentOS7home:/home : 공유할 볼륨 경로. : 기준으로 앞이 로컬 볼륨 경로, 뒤가 컨테이너 내부 경로. 변경/생략 가능.
	
					-p 44983:22 : 개방 포트 설정. 외부에서 44983 포트로 접근하면 컨테이너의 22번 포트로 연결 (ssh 포트: 22). 변경, 생략 가능.
	
					centos:centos7 : 컨테이너의 베이스 이미지 이름. 다른 리눅스 이미지를 원할 경우 교체.
